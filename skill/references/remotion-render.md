# remotion-render.md: render path B (Remotion free-compose)

Path B swaps the HTML+Playwright render layer for Remotion (React, animated per
frame). THE BRAIN IS UNCHANGED: Phase 0 intake, Phase 1 deep extract, Phase 2
narrative plan (PA1 arc, TTS rules 2.2.6, caveat-as-feature, pain-CTA outro) apply
exactly as written in SKILL.md. Only Phases 3-5 run differently.

When to use path B: the user says "render with remotion" / "--remotion", or the
video needs animation beyond the CSS ceiling (live counters, self-drawing charts,
springs, animated connectors, word-level karaoke rendered directly). Path A (HTML
templates) stays the default.

Design doctrine lives in `render-remotion/SCENE-DESIGN.md`. Read it before
composing: it is the law for scene structure, block density, safe zones and every
paid-for visual lesson.

## Layout

```
render-remotion/
  package.json            # remotion 4.0.484 + @remotion/media + google-fonts
  SCENE-DESIGN.md         # scene design doctrine (READ FIRST)
  docs/skins/             # visual gallery of the 13 skins
  src/lib/core.tsx        # Rise/Pop/SceneFade, countAt/progressAt, clipLineToRect, types
  src/lib/karaoke.tsx     # KaraokeNeon / KaraokeMarker / KaraokeSticker (word-level)
  src/lib/skins.ts        # 14 verified skins (tokens, NOT layouts)
  src/videos/<slug>.tsx   # ONE FILE PER RUN: the agent composes bespoke scenes here
  src/videos/<slug>-data.ts  # generated by tools/gen_voice.py, never hand-edit
  src/Root.tsx            # one <Composition> registered per video
  tools/gen_voice.py      # TTS word-timestamps + phonetic merge (edge-tts)
  tools/gen_voice_google.py  # Google Chirp3 HD variant (estimated word timing)
  tools/gen_voice_vieneu.py  # VieNeu-TTS local variant (free, voice cloning, bilingual <en> tags)
  tools/vieneu_bridge.py     # runs INSIDE the VieNeu uv env (spawned by gen_voice_vieneu)
  public/audio/           # per-scene mp3 (gitignored)
  public/footage/         # Playwright/product footage mp4 (gitignored)
```

## Per-run process (replaces path A Phases 3-5)

### B3. Voice + word timestamps

From the Phase 2 plan, write `narration.json` = `[{"id": "<sceneId>", "narration": "..."}]`
(narration ALREADY follows rule 2.2.6: numbers spelled out, terms transliterated,
no URLs/emoji), then:

```bash
python tools/gen_voice.py --scenes <run>/narration.json --prefix <slug> \
    --data-out src/videos/<slug>-data.ts
```

- Default voice: `vi-VN-NamMinhNeural`, rate `+15%` (female alt: `vi-VN-HoaiMyNeural`).
- Three voice paths, pick per user request:
  - `tools/gen_voice.py` = edge-tts (free, real WordBoundary timestamps). Apply the
    phonetic spellings from rule 2.2.6 d.2 in full.
  - `tools/gen_voice_google.py` = Google Chirp3 HD (paid; default
    vi-VN-Chirp3-HD-Charon, northern male, rate +5%). Google returns NO word
    boundaries, so karaoke timing is ESTIMATED from syllable weights plus
    punctuation pauses (smooth enough in practice).
    Phonetics on Chirp: write tricky acronyms as ONE HYPHENATED token so Chirp
    reads them as a unit ("ây-ai" for AI, "ây-pi-ai" for API, "git-hâb" for
    GitHub). Two separate tokens ("ây ai") get read with a split beat, and raw
    "AI" gets letter-spelled in Vietnamese. Names containing digits always need
    phonetics (any2video = "en ni tu vi đeo"). Captions always display the clean
    spelling.
  - `tools/gen_voice_vieneu.py` = VieNeu-TTS (local, free, CPU-friendly, instant
    voice cloning). Point --vieneu-dir or the VIENEU_TTS_DIR env var at a local
    VieNeu-TTS checkout; the heavy work runs inside that env via `uv run --no-sync`.
    Voice: `--ref-audio <3-8s clip>` (clone) or `--voice <preset>`; delivery via
    `--style` (default tu_nhien) + `--temperature`. Karaoke timing is ESTIMATED
    like the Google path. BILINGUAL (sea-g2p code-switching): write English terms
    RAW with correct spelling/casing ("GitHub", "AI", "Claude Code") and they read
    as native English. Do NOT use `<en>` markup (unsupported; a tagged acronym like
    "AI" flips to the Vietnamese word "ai") and do NOT use edge-style phonetics
    ("git hâb" gets its diacritic names spelled out). Names containing digits still
    need spelling out ("any to video" for any2video, else "2" reads as Vietnamese
    "hai") + a PHONETIC_MAP merge so captions show the clean name. When unsure how
    a term will read, inspect phonemes first:
    `uv run --no-sync python -c "import sys; sys.path.insert(0,'src'); from vieneu_utils.phonemize_text import phonemize_text; print(phonemize_text('cau thu nghiem'))"`.
- edge-tts 7.x REQUIRES `boundary="WordBoundary"` (the script sets it; the lib
  default SentenceBoundary returns 0 words).
- PHONETIC_MAP in gen_voice.py merges phonetic clusters back into clean caption
  words. Add new clusters to the map when a script introduces new terms.
- Scene duration = audio + pad (default 0.8s, tune with `--pad` to hit the length
  target), min 100 frames. The printed TOTAL must land in the 50-75s window or the
  user's requested target (SKILL.md 2.2.6 b.2). Reconcile overshoot by trimming
  pad or the wordiest narrations, NEVER by touching approved wording first.

### B4. Compose scenes (replaces the Phase 4 template step)

There is NO template pool. Each scene is designed in 3 steps
(full doctrine: `render-remotion/SCENE-DESIGN.md`):

1. **Pick ONE skin for the whole video** (never mix): browse `docs/skins/` and
   `src/lib/skins.ts`, choose by content mood. A skin is tokens (colors/fonts/
   shape language/karaoke variant); you write the styles yourself from those
   tokens. **GitHub repo tours default to `repo-dark`** (tech-dark-neon blended
   with escbase-starfield: glow cards + starfield + badge pills + stat rows)
   unless the user picks another skin. Preview stills regenerate via the
   `skin-gallery` composition (`npx remotion still skin-gallery --frame=<N>`).
2. **For each scene ask: "what STRUCTURE does this narration have?"**
   (convergence, comparison, flow, cadence, ratio, ranking, timeline,
   transformation, filter, division of labor...) then draw that structure in
   plain JSX/SVG. Labels 3 words max. Never repeat a layout within a video,
   never paste a layout from an earlier video.
3. **The slide is the LABEL, the voice is the COMMENTARY: the two channels never
   share text.** Karaoke already carries the transcript; do not write the
   narration onto the scene.

Write `src/videos/<slug>.tsx`: import `SCENES`/`TOTAL_FRAMES` from
`<slug>-data.ts`, map scene ids to body components, each `<Sequence>` holds
body + karaoke + `<Audio>` (from `@remotion/media`). Register the composition in
`src/Root.tsx` (id = slug, 1080x1920, fps 30). Complete living examples:
`src/videos/flow-movie-pipeline-pop.tsx`,
`src/videos/flow-movie-pipeline-tour.tsx`,
`flow-movie-pipeline-repodark.tsx` (the tour re-skinned to repo-dark on the same
data file: how to swap skins without touching script or audio),
`chanktb-any2video.tsx` (the tool's own intro: VieNeu cloned voice, real skin
montage via docs/skins PNGs, hybrid footage), and `fb-ads-2026.tsx` (a NEWS video
from a fact-checked article, escbase-starfield skin).

Hard rules while composing (inherited from path A + paid-for lessons):
- Animate ONLY with `interpolate()`/`spring()` on the current frame. CSS
  animations/transitions are FORBIDDEN.
- No hooks inside `.map()`: use `countAt`/`progressAt` (lib) with the frame read
  once.
- Connectors must STOP AT THE BLOCK EDGE: use `clipLineToRect` (lib); never draw
  a line through a panel.
- Safe zone: content x 90-990, y 345-1575. Karaoke band y 1470-1635: scenes keep
  it empty.
- Stickers/chips outside a headline must set an explicit fontSize (they inherit
  a 16px root otherwise).
- Numbers on slides as digits, numbers in narration spelled out. No em-dash, no
  italics for Vietnamese, Vietnamese line-height 1.22+.
- Fonts via `@remotion/google-fonts/<Font>` with subsets `["latin", "vietnamese"]`;
  tsc blocks fonts lacking the vietnamese subset (DMMono/SpaceGrotesk are
  latin-only: digits/EN labels exclusively).

### B4.5 Real footage (HYBRID with path A, for the repo-scroll and outro beats)

The two footage beats in the PA1 arc (repo scroll after the reveal, author-profile
outro) are NOT drawn in Remotion. Use path A's Playwright tool as-is and embed the
mp4 in the timeline:

```bash
# duration = exactly the audio duration of that scene (printed by gen_voice.py)
python -m lib.render.repo_footage --url https://github.com/<owner>/<repo> \
    --out render-remotion/public/footage/<slug>_scroll.mp4 --duration <seconds>
python -m lib.render.repo_footage --url https://github.com/<owner> \
    --out render-remotion/public/footage/<slug>_author.mp4 --duration <seconds>
```

If the repo is PRIVATE, a repo scroll is impossible (404) and every star/repo-link
CTA is wrong: swap the scroll beat for real product footage (a clip the tool
produced) and point the outro CTA at follow/inbox plus the author profile instead.

In the video file: `<FootageScene src="footage/<slug>_scroll.mp4" />` (from
`src/lib/footage.tsx`) + karaoke + `<Audio>` like any scene. Footage comes with
the browser bar + bottom scrim baked in, so karaoke stays readable on top. Living
demo: the `footage-demo` composition (regen commands in that file's comments).

### B5. Gates + render (replaces the Playwright ship-gate)

```bash
npx tsc --noEmit                                   # gate 1: types + font subsets
npx remotion still <slug> --frame=<mid-scene>      # gate 2: inspect EVERY scene
npx remotion render <slug> out/<slug>.mp4          # full render
ffprobe out/<slug>.mp4                             # verify duration
ffmpeg -i out/<slug>.mp4 -af volumedetect -f null - # verify voice (mean ~-20dB, max < -1dB)
```

Gate 2 is mandatory: render a still mid-scene for EVERY scene, inspect for layout
bugs (overflow, overlap, ugly wraps, tiny stickers) and fix BEFORE the full
render. This is the cheap, fast equivalent of path A's scene_gate.

### B6. BGM + delivery

BGM uses path A's mixer unchanged, as a post step on the rendered mp4 (ducked
under the voice; pick a track via `resolve_bgm` or point straight at a file in
`skill/templates/bgm/`; credit CC-BY tracks in the post):

```bash
python -m lib.audio.bgm mix render-remotion/out/<slug>.mp4 \
    skill/templates/bgm/<track>.mp3 --out workspace/runs/<slug>/final.mp4
```

Then prepend a 0.1s poster frame: Remotion scenes fade in from black, so frame 0
is black and the feed thumbnail dies without it. Render a still of the hook
scene's peak frame as the poster, then:

```bash
ffmpeg -loop 1 -t 0.1 -i poster.png -i final_nobump.mp4 -filter_complex \
  "[0:v]scale=1080:1920,fps=30,format=yuv420p[p];[p][1:v]concat=n=2:v=1:a=0[v];[1:a]adelay=100|100[a]" \
  -map "[v]" -map "[a]" -c:v libx264 -crf 19 -c:a aac -b:a 192k final.mp4
```

Everything else matches path A: ship_gate on the final file, Telegram checkpoints
if intake enabled them, post caption carries the repo URL (the video never speaks
a URL, outro rule 2.2.5.0).
